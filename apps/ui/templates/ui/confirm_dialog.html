{% load i18n ui %} {% comment %} Global confirmation dialog. Mounted once in the shell; any form that carries a `data-confirm` attribute is routed through it instead of the browser's confirm().
{# optional, default danger #} Works for a plain POST and an HTMX form alike: the submit is intercepted in the capture phase (before HTMX's own handler), the dialog opens, and on confirm the form is re-submitted with a flag set so this handler steps aside. {% endcomment %}
